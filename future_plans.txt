  [핵심 확장] 대본 PDF → 캐릭터 분석 → 캐릭터 반영 스캔션
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[개요]
뮤지컬 대본 PDF를 업로드하면 AI가 각 캐릭터를 자동 분석하여,
해당 캐릭터의 성향/말투/감정을 반영한 윤색을 수행하는 기능.

[처리 흐름]

1) 대본 PDF 업로드 & 파싱
   - PDF → 텍스트 추출 (PyMuPDF, pdfplumber)
   - 대사/지문/무대지시 자동 분류
   - 캐릭터별 대사 분리

2) 캐릭터 분석 (LLM 기반)
   - 각 캐릭터의 대사를 모아서 분석
   - 추출 항목:
     * 성격 특성 (personality traits)
     * 말투 스타일 (formal/informal, 존댓말/반말, 사투리 등)
     * 감정 아크 (장면별 감정 변화)
     * 교육 수준 / 사회적 지위 (어휘 수준에 영향)
     * 다른 캐릭터와의 관계 (누구에게 어떻게 말하는지)
     * 반복 사용하는 표현/구절 (캐릭터 시그니처)

3) 캐릭터 프로필 생성
   - JSON 형태의 캐릭터 프로필
   - 사용자가 검토/수정 가능

   예시:
   {
     "name": "Phantom",
     "personality": "집착적, 천재적, 고독한",
     "speech_style": "시적이고 극적인 표현, 명령형 자주 사용",
     "register": "높은 어휘 수준, 격식체",
     "emotion_arc": {
       "Act1": "분노와 갈망",
       "Act2": "절망과 체념"
     },
     "signature_phrases": ["밤의 음악", "내 음악의 천사"],
     "relationship_speech": {
       "Christine": "부드럽지만 소유욕 강한 어조",
       "Raoul": "경멸적이고 공격적인 어조"
     }
   }

4) 캐릭터 반영 윤색
   - 기존 스캔션 제약조건 + 캐릭터 프로필을 LLM에 전달
   - 같은 의미라도 캐릭터에 따라 다른 어휘/말투 선택
   - 예: "사랑해" →
     * 순수한 캐릭터: "널 좋아해"
     * 극적인 캐릭터: "내 영혼이 너를 원해"
     * 코믹한 캐릭터: "아 나 너한테 빠졌나봐"

[기술 요구사항]
- PDF 파싱: PyMuPDF 또는 pdfplumber
- 대본 구조 분석: LLM (캐릭터명/대사/지문 분류)
- 캐릭터 분석: Claude Opus (복잡한 분석이므로 고성능 모델)
- 프로필 에디터: Streamlit JSON 에디터 또는 별도 UI

[개발 우선순위: Phase 2 — 핵심 스캔션 완성 후 진행]
예상 추가 개발: 4~6주
